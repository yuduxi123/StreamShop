任务拆解
工程架构
- 移动端优先： 实现一个移动客户端应用，可选择 React Native、Flutter、原生 Android/iOS、小程序或移动端 H5。
- 前后端分离： 后端使用 Node.js 提供 API 服务，负责用户、视频、商品、购物车、订单、互动数据等业务逻辑。
- 代码仓库： 项目整体代码存放于单一 Git 仓库，方便管理和协同。
- 编码规范： 集成 ESLint、Prettier、Husky 等工具，确保代码风格统一和质量。
- 结构清晰： 要求代码分层合理，模块化清晰，具备良好的可读性和可维护性。
核心功能
用户端移动客户端
- 内容流浏览：
  - 实现短视频或直播间列表，支持上下滑切换内容。
  - 每个内容页展示视频、主播/作者信息、点赞、评论、分享、商品入口等基础元素。
  - 支持视频播放、暂停、静音、进度展示等基础能力。
- 直播/视频商品卡：
  - 在视频播放区域展示商品浮层卡片，包括商品图、标题、价格、优惠信息、库存状态等。
  - 点击商品卡后，弹出商品详情半屏浮层，用户可以查看规格、优惠、评价摘要等信息。
  - 支持用户将商品加入购物车或立即购买。
- 互动能力：
  - 支持点赞、评论、收藏视频或商品。
  - 支持直播间弹幕或实时评论列表。
  - 支持模拟在线人数、热度值、商品讲解中的“正在讲解”状态。
- 购物链路：
  - 实现购物车页面，支持商品数量修改、删除、勾选结算。
  - 实现订单确认页，展示商品信息、优惠金额、收货地址、实付金额等。
  - 实现模拟下单和支付结果页，支付成功后可查看订单详情。
  - 实现订单列表，支持按待支付、待发货、已完成等状态筛选。
商家/运营后台
- 内容管理：
  - 支持上传或配置短视频内容，包括标题、封面、视频地址、关联商品等。
  - 支持设置视频状态，如草稿、已发布、已下架。
- 商品管理：
  - 支持商品列表展示，包含商品名称、价格、库存、销量、状态等字段。
  - 支持新增、编辑、上下架商品。
  - 支持将商品绑定到指定短视频或直播间。
- 直播间配置：
  - 支持创建模拟直播间，配置直播标题、封面、主播信息、商品列表。
  - 支持设置当前讲解商品，并将状态同步到用户端。
后端服务
- RESTful API：
  - 提供用户、视频、商品、购物车、订单、评论等核心 API。
  - 支持内容列表分页、商品查询、购物车增删改查、订单创建等能力。
- 实时通信：
  - 使用 WebSocket 或类似技术，实现直播间评论、弹幕、在线人数、当前讲解商品等实时推送。
  - 用户端需要能够实时接收并更新直播间状态。
- 数据持久化：
  - 使用 SQLite、MySQL、PostgreSQL、MongoDB 或 JSON 文件等方式存储业务数据。
  - 至少保证商品、视频、购物车、订单、评论数据可持久化。
用户体验与客户端挑战
- 视频播放体验： 视频切换要尽量流畅，避免明显白屏、卡顿或重复加载。
- 移动端交互设计： 商品浮层、评论区、购物车、下单页需要符合移动端操作习惯。
- 性能优化： 内容流需要支持分页加载、懒加载、缓存，避免长列表滑动卡顿。
- 弱网体验： 视频加载、接口请求失败、下单失败等场景需要有清晰反馈。
- 状态同步： 直播间商品讲解、弹幕、评论、库存变化等实时数据需要与界面状态保持一致。
- 多端适配： 适配不同屏幕尺寸，重点关注小屏手机下的内容遮挡、按钮误触和浮层高度问题。
加分项
数据可视化看板
- 设计一个运营分析页面，展示内容和商品转化数据。
- 可展示维度：
  - 视频播放量、点赞量、评论量趋势。
  - 商品曝光、点击、加购、下单转化漏斗。
  - 不同视频或直播间的 GMV 排行。
  - 热门商品 Top 10。
智能商品推荐
- 根据用户浏览、点赞、收藏、加购行为，推荐相关商品。
- 在视频详情或购物车页面展示“猜你喜欢”商品列表。
- 可简单实现基于标签、类目、价格区间的推荐逻辑。
优惠与营销玩法
- 支持优惠券、限时折扣、满减、秒杀倒计时等营销组件。
- 在直播间内实时推送优惠券，并支持用户领取。
- 订单结算时自动计算优惠金额。
高级视频体验
- 支持视频预加载，提升上下滑切换体验。
- 支持直播间小窗播放。
- 支持商品讲解片段定位，点击商品后跳转到对应视频时间点。
AIGC 能力
- 支持商家输入商品信息后，自动生成短视频标题、商品卖点或直播讲解文案。
- 支持根据商品图和描述生成商品推荐语。
FAQ
1. 技术栈必须严格遵守吗？不强制。移动端可以选择 React Native、Flutter、原生 Android/iOS、小程序或移动端 H5。后端推荐使用 Node.js 和 TypeScript，但也可以选择自己更熟悉的方案。
2. 必须实现真实直播吗？不必须。可以通过本地视频、模拟直播状态和 WebSocket 实时消息来完成直播间体验。评审更关注业务闭环、交互体验和工程质量。
3. 支付能力需要接入真实支付吗？不需要。可以实现模拟支付流程，包括订单创建、支付中、支付成功、支付失败等状态。
4. 视频素材从哪里来？可以使用公开测试视频、本地 mock 视频，或自己录制简单商品讲解视频。重点不在视频内容本身，而在移动端播放、互动和购物链路。
5. 可以组队吗？可以，建议 2-3 人组队。可以按移动客户端、后端服务、运营后台和数据可视化进行分工。
